Subscription Service API
REST-сервис для агрегации данных об онлайн-подписках пользователей

🏗 Структура проекта
Проект реализован с соблюдением принципов чистой архитектуры (Clean Architecture):

cmd/app/main.go — точка входа в приложение, инициализация конфигов, логгера и зависимостей

internal/handler — слой доставки (HTTP/REST). Принимает запросы, валидирует входные данные

internal/service — слой бизнес-логики. Здесь описываются правила работы сервиса

internal/repository — слой данных. Работа с базой данных через GORM

internal/model — описание структур данных (сущности БД)

internal/config — загрузка конфигураций из .env

migrations/ — SQL-файлы для ручного управления структурой базы данных

docs/ — автоматически сгенерированная Swagger-документация

🚀 Запуск и развертывание
Файлы Dockerfile и docker-compose.yml намеренно вынесены в корневую директорию проекта. Это сделано для удобного и быстрого развертывания всей инфраструктуры одной командой без необходимости спускатьтся к папке

Команды для запуска:
С помощью Docker Compose:
docker-compose up --build -d

Или с помощью Makefile :
make docker-up

🗄 База данных и миграции
В проекте реализовано два подхода к управлению схемой БД:

Автомиграция (GORM AutoMigrate): Настроена в main.go для быстрого старта. При запуске сервиса база автоматически создаст необходимые таблицы.

Обычные миграции: В папке /migrations подготовлены SQL-скрипты схемы данных.

📖 Документация API
После запуска сервиса интерактивная документация Swagger доступна по адресу: http://localhost:8080/swagger/index.html

Через интерфейс Swagger можно протестировать:

Создание подписки (POST).

Получение списка подписок (GET).

Расчет суммарной стоимости подписок за период с фильтрацией по пользователю (UUID) и сервису.
